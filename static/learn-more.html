FinShield Fraud Detection Architecture

FinShield is an explainable fraud detection system that analyzes transaction behavior, device identity, login patterns and risk signals to detect fraudulent financial activity in real time.

System Workflow

User Authentication

Users login through Firebase authentication. Each transaction is tied to a verified identity.

Evidence Collection

System automatically captures: IP Address Device fingerprint Transaction timing Behavior patterns

Feature Engineering

Raw data is converted into ML features: IP change flag Device change flag Transaction velocity Behavior deviation

ML Risk Prediction

Random Forest model calculates fraud probability using 10 behavioral features.

Explainable Output

System shows: Risk score Evidence data Risk factors Safe factors

Risk Indicators Used

  • IP address change
  • Device fingerprint change
  • Transaction time anomaly
  • Transaction frequency spike
  • Account age risk
  • Failed login attempts
  • Transaction pattern deviation
  • Amount anomaly detection

Why This System Is Realistic

Unlike basic academic demos, FinShield does not rely on manual fraud flags. It uses automatic behavioral analysis similar to real banking fraud detection engines.

Behavior signals → Risk features → ML prediction → Explainable decision

Technology Stack

  • FastAPI backend
  • Firebase authentication
  • Firestore database
  • Random Forest ML model
  • Explainable fraud scoring UI
  • Behavior risk analysis engine

System Output

Each transaction produces:

  • Fraud / Normal decision
  • Risk percentage
  • Risk severity level
  • Behavior evidence
  • Safe indicators
  • Fraud reasoning
